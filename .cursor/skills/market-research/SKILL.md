# 市場調査 (Market Research)

意思決定を支えるリサーチを作成する。リサーチのためのリサーチ（research theater）ではない。

## 有効化タイミング (When to Activate)

- 市場、カテゴリ、企業、投資家、技術トレンドを調査するとき
- TAM/SAM/SOM 推定を構築するとき
- 競合または隣接プロダクトを比較するとき
- アウトリーチ前の投資家ドシエを準備するとき
- 構築、資金調達、市場参入前にテーゼを圧力テストするとき

## リサーチ基準 (Research Standards)

1. 重要な主張にはすべて出典が必要。
2. 最新データを優先し、古いデータは明示する。
3. 反対意見の証拠とダウンサイドケースを含める。
4. 要約だけでなく、判断に落とし込む。
5. 事実、推論、推奨を明確に分離する。

## 一般的なリサーチモード (Common Research Modes)

### 投資家 / ファンドデューデリジェンス (Investor / Fund Diligence)
収集項目:
- ファンド規模、ステージ、典型的なチェックサイズ
- 関連ポートフォリオ企業
- 公開テーゼと最近の活動
- ファンドが適合する／しない理由
- 明らかなレッドフラグやミスマッチ

### 競合分析 (Competitive Analysis)
収集項目:
- マーケティングコピーではなくプロダクトの実態
- 公開されている場合の資金調達・投資家履歴
- 公開されている場合のトラクション指標
- 流通・価格設定の手がかり
- 強み、弱み、ポジショニングのギャップ

### 市場規模 (Market Sizing)
使用手法:
- レポートや公開データセットからのトップダウン推定
- 現実的な顧客獲得前提によるボトムアップ妥当性チェック
- 論理の各飛躍に明示的な前提

### 技術 / ベンダーリサーチ (Technology / Vendor Research)
収集項目:
- 仕組み
- トレードオフと採用シグナル
- 統合の複雑さ
- ロックイン、セキュリティ、コンプライアンス、運用リスク

## 出力形式 (Output Format)

デフォルト構成:
1. executive summary
2. key findings
3. implications
4. risks and caveats
5. recommendation
6. sources

## 品質ゲート (Quality Gate)

納品前に確認:
- すべての数値に出典があるか、推定と明記されているか
- 古いデータにフラグが付いているか
- 推奨が証拠から導かれているか
- リスクと反論が含まれているか
- 出力が意思決定を容易にしているか
