# ECC Hook 修正 — 2026-04-21 (ECC Hook Fix — 2026-04-21)

## 概要 (Summary)

Windows 上の Claude Code CLI v2.1.116 で、すべての Bash tool hook 呼び出しが次のエラーで失敗していました：

```
PreToolUse:Bash hook error
Failed with non-blocking status code:
C:\Program Files\Git\bin\bash.exe: C:\Program Files\Git\bin\bash.exe:
cannot execute binary file

PostToolUse:Bash hook error  (同上)
```

結果：`observations.jsonl` は `2026-04-20T23:03:38Z` 以降更新が止まりました
（最後のエントリは以前の stdin BOM 問題による `parse_error`）。

## 根本原因 (Root Cause)

`C:\Users\sugig\.claude\settings.local.json` に2つの欠陥がありました：

### 欠陥 1 — UTF-8 BOM + CRLF 改行 (Defect 1 — UTF-8 BOM + CRLF line endings)

ファイルは `EF BB BF`（UTF-8 BOM）で始まり、`CRLF` 改行を使用していました。
これは PowerShell の `ConvertTo-Json | Out-File` のデフォルト動作であり、
`patch_settings_cl_v2_simple.ps1` がファイルを書き換えるときに残るものです。

```
00000000: efbb bf7b 0d0a 2020 2020 2268 6f6f 6b73  ...{..    "hooks
```

### 欠陥 2 — 二重ラップされた bash.exe 呼び出し (Defect 2 — Double-wrapped bash.exe invocation)

command 文字列が明示的に bash.exe を再呼び出ししていました：

```json
"command": "\"C:\\Program Files\\Git\\bin\\bash.exe\" \"C:\\Users\\sugig\\.claude\\skills\\continuous-learning\\hooks\\observe-wrapper.sh\""
```

Claude Code が Windows でこれを spawn するとき、引数分割が
引用符付き `"C:\Program Files\..."` トークンを正しく保持しません。
`Program Files` 内のスペースが `argv[0]` を分割し、`bash.exe` が
スクリプトファイルとして自分自身に渡され、次が発生します：

```
bash.exe: bash.exe: cannot execute binary file
```

### 以前の動作していた形（参考）(Prior working shape (for reference))

`patch_settings_cl_v2_simple.ps1` 実行前、command は単純でした：

```json
"command": "C:\\Users\\sugig\\.claude\\skills\\continuous-learning\\hooks\\observe.sh"
```

Windows 上の Claude Code は `.sh` を検出し、手動の `bash.exe` ラップなしで
Git Bash 経由で呼び出します。

## 修正 (Fix)

`C:\Users\sugig\.claude\settings.local.json` を UTF-8（BOM なし）、LF
改行で書き換え、command を wrapper `.sh` に直接指し、
hook phase をプレーン引数として渡すようにしました：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "C:/Users/sugig/.claude/skills/continuous-learning/hooks/observe-wrapper.sh pre"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "C:/Users/sugig/.claude/skills/continuous-learning/hooks/observe-wrapper.sh post"
          }
        ]
      }
    ]
  }
}
```

副次効果：`pre` / `post` 引数が `observe.sh` の
`HOOK_PHASE` 変数にルーティングされ、イベントが `tool_start` と
`tool_complete` として正しく記録されます（以前はすべて `tool_complete` として記録されていました）。

## 検証 (Verification)

両 hook phase をエミュレートして、新しい command 形式を直接呼び出し：

```bash
# PostToolUse path
echo '{"tool_name":"Bash","tool_input":{"command":"pwd"},"session_id":"post-fix-verify-001","cwd":"...","hook_event_name":"PostToolUse"}' \
  | "C:/Users/sugig/.claude/skills/continuous-learning/hooks/observe-wrapper.sh" post
# exit=0

# PreToolUse path
echo '{"tool_name":"Bash","tool_input":{"command":"ls"},"session_id":"post-fix-verify-pre-001","cwd":"...","hook_event_name":"PreToolUse"}' \
  | "C:/Users/sugig/.claude/skills/continuous-learning/hooks/observe-wrapper.sh" pre
# exit=0
```

`observations.jsonl` に追加されました：

```
{"timestamp":"2026-04-21T05:57:54Z","event":"tool_complete","tool":"Bash","session":"post-fix-verify-001",...}
{"timestamp":"2026-04-21T05:57:55Z","event":"tool_start","tool":"Bash","session":"post-fix-verify-pre-001","input":"{\"command\":\"ls\"}",...}
```

両 phase が正しい型のイベントを生成するようになりました。

**ライブ CLI 検証に関する注記:** settings 変更は次の
`claude` CLI セッション起動時に有効になります。CLI を再起動し Bash tool 呼び出しを実行して、
実際の CLI セッションから `observations.jsonl` に新しい行が現れることを確認してください。
