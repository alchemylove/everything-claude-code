## 所見 (Findings)

### Medium

1. `skills/continuous-learning-v2/hooks/observe.sh`

新しい `CLAUDE_CODE_ENTRYPOINT` guard は既知の非 `cli` 値の有限 allowlist
（`sdk-ts`、`sdk-py`、`sdk-cli`、`mcp`、`remote`）を使います。

将来互換性の穴を残します：将来の非 `cli` entrypoint 値は
フォールスルーし、対話型として扱われます。PR が防ごうとする
自動セッション観測のクラスを再導入します。

より安全なルールは：

- `cli` のみ許可
- 他の明示的 entrypoint をすべて automated として扱う
- 変数が未設定のときのデフォルトフォールバックは `cli` のまま

推奨形状：

```bash
case "${CLAUDE_CODE_ENTRYPOINT:-cli}" in
  cli) ;;
  *) exit 0 ;;
esac
```
